## A. 微信支付（不配则下单付款报“微信支付暂未配置”）

读取位置：`cicada-client-order` 的 `getWechatPayConfig()` / `getWechatPayApiV3Key()`。

设置位置：uniCloud Web 控制台 https://unicloud.dcloud.net.cn → 选服务空间 **doctor-app** → 云函数 → **环境变量 / 公共配置**（让 `cicada-client-order` 能读到 `process.env`）。

| 环境变量名 | 值 / 来源 | 必填 |
|---|---|---|
| `WX_PAY_APPID` | 小程序 AppID `wx25289fbe4a3bf011`（也可复用 `WX_APPID`） | ✅ |
| `WX_PAY_MCH_ID` | 微信支付商户号（商户平台 → 账户中心 → 商户号） | ✅ |
| `WX_PAY_SERIAL_NO` | 商户 API 证书序列号（商户平台 → API 安全 → API 证书） | ✅ |
| `WX_PAY_PRIVATE_KEY` | 商户 API 私钥 `apiclient_key.pem` 全文（含 `-----BEGIN/END-----`，换行用 `\n`）<br>或 `WX_PAY_PRIVATE_KEY_BASE64` 填该 pem 的 base64 | ✅（二选一） |
| `WX_PAY_API_V3_KEY` | APIv3 密钥（商户平台 → API 安全 → APIv3 密钥，32 位） | ✅（支付回调解密用） |
| `WX_PAY_NOTIFY_URL` | 支付结果回调 URL，见下方 A-1 | ✅ |

### A-1. 回调 URL（WX_PAY_NOTIFY_URL）
1. uniCloud 控制台为 `cicada-client-order` 开启 **云函数 URL 化**，得到形如 `https://xxx.bspapp.com/http/cicada-client-order` 的地址。
2. `WX_PAY_NOTIFY_URL` 填指向 `wechatPayNotify` 方法的 URL 化地址（该方法已写好，靠 `this.getHttpInfo()` 解析微信回调并解密 → 以服务端查单结果为准）。
3. 在**微信支付商户平台**把该域名加入 Native/JSAPI 支付回调域名白名单。

> 也支持 `WXPAY_*` / `WECHAT_PAY_*` 前缀（同名取其一即可），但建议统一用 `WX_PAY_*`。

## B. 微信订阅消息（不配则提醒静默跳过，写一条 skipped 日志，不报错）

### B-1. 公众号/小程序基础（发订阅消息要先拿 access_token）
读取位置：`getWechatAppConfig()`。

| 环境变量名 | 值 |
|---|---|
| `WX_APPID` | `wx25289fbe4a3bf011` |
| `WX_SECRET` | 小程序密钥（微信公众平台 → 开发管理 → 开发设置 → AppSecret） |

### B-2. 7 个订阅模板 ID
在**微信公众平台 → 订阅消息**申请一次性订阅模板，拿到每个的模板 ID，按下表填环境变量（命名规则 `WX_SUBSCRIBE_TEMPLATE_<场景大写>`）：

| 环境变量名 | 触发场景 |
|---|---|
| `WX_SUBSCRIBE_TEMPLATE_REPAIR_SUBMITTED` | 报修已提交 |
| `WX_SUBSCRIBE_TEMPLATE_ORDER_RECEIVED` | 设备已签收 |
| `WX_SUBSCRIBE_TEMPLATE_QUOTE_ISSUED` | 维修报价已发布 |
| `WX_SUBSCRIBE_TEMPLATE_PAYMENT_CONFIRMED` | 付款已确认 |
| `WX_SUBSCRIBE_TEMPLATE_ORDER_SHIPPED` | 设备已回寄 |
| `WX_SUBSCRIBE_TEMPLATE_ORDER_COMPLETED` | 工单完成 |
| `WX_SUBSCRIBE_TEMPLATE_REVIEW_INVITE` | 服务评价邀请（本轮新增） |

> 模板 ID 配在哪个云函数：`cicada-client-order`（报修/支付/完成/回访）与 `cicada-admin-order`（后台改状态触发）都要能读到，建议设为**服务空间全局环境变量**。

## C. 数据库索引（量大前必须建，否则慢/可能重号）

设置位置：uniCloud Web 控制台 → 数据库 → 选集合 → **索引管理**。完整清单见 `uniCloud-alipay/database/INDEXES.md`，**本轮务必确认这几条**：

- `cicada_orders.order_no` —— **唯一索引**（防重号，最关键）
- `cicada_orders` 复合：`customer_id, create_time desc`（身份桥后台查历史工单）
- `cicada_customers`：`user_id`、`openid`、`phone`（身份桥下单匹配/建档）
- `cicada_user_devices`：`user_id, sn`、`customer_id, create_time desc`、`sn`（设备台账合流）
- `cicada_subscription_logs`：`scene, create_time desc`、`status, create_time desc`（排查订阅推送）

> 复合索引：在同一面板多次「添加字段」后一起保存。

## E. 配完自检

1. **支付**：小程序提交一单 → 后台发报价 → 客户「确认并支付」走通微信支付 → 工单自动进维修中、`cicada_subscription_logs` 有 `payment_confirmed` 记录（非 skipped）。
2. **身份桥**：用一个**新**手机号小程序下单 → 后台「客户管理」应自动出现该客户（source=miniapp）→ 点进去能看到这张工单。
3. **回访**：把一单推到已完成 → 客户「确认收货/去评价」→ `cicada_subscription_logs` 出现 `order_completed` 和 `review_invite`（配了模板 ID 则 sent，否则 skipped）。
4. **设备**：报修带 SN → 「我的设备」出现该设备；后台对应客户的设备台账里也能看到（customer_id 已回填）。
