# 微信支付 + 一键开票（微信电子发票/服务商）上线配置清单

> 方案：**微信支付收款 → 财务确认到账 → 后台「一键开票」→ 开票服务商自动开票 + 回填链接/号码/日期 + 推微信电子发票卡包**。
> 电子税务局手动开票（上传/回填）保留作专票/红冲/特殊抬头的兜底。

---

## 一、微信支付（前提，必须先打通）

微信电子发票挂在微信支付上，所以**先把微信支付配好**。支付逻辑在 `cicada-client-order`（下单/JSAPI 支付）和 `cicada-admin-order`（退款/核销）两个云函数。

### 1. 微信支付商户平台要做的
- 开通 **JSAPI 支付**（小程序支付）
- 关联小程序 AppID：`wx25289fbe4a3bf011`
- 申请/下载 **商户 API 证书**（apiclient_cert.pem / apiclient_key.pem），记下 **证书序列号**
- 设置 **APIv3 密钥**
- 配置支付结果回调 URL（指向云函数 URL 化地址）

### 2. 云函数环境变量（`cicada-client-order` + `cicada-admin-order` 都要配）
| 环境变量 | 说明 |
|---|---|
| `WX_PAY_APPID`（或 `WX_APPID`） | 小程序 AppID `wx25289fbe4a3bf011` |
| `WX_PAY_MCH_ID` | 微信支付商户号 |
| `WX_PAY_SERIAL_NO` | 商户 API 证书序列号 |
| `WX_PAY_PRIVATE_KEY`（或 `WX_PAY_PRIVATE_KEY_BASE64`） | 商户 API 私钥（apiclient_key.pem 内容；BASE64 版更稳，避免换行丢失） |
| `WX_PAY_API_V3_KEY` | APIv3 密钥 |
| `WX_PAY_NOTIFY_URL` | 支付结果回调地址（云函数 URL 化） |
| `WX_PAY_REFUND_NOTIFY_URL` | 退款回调地址（可选） |

> 缺任一项，云函数会抛「微信支付暂未配置：…」。配置位置：uniCloud 控制台 → 对应云函数 → 环境变量。

### 3. 验证
- 小程序发起一笔支付能拉起微信收银台、支付成功、回调把 `payment_status` 置为已到账。

---

## 二、微信电子发票（客户侧渠道）

- 微信支付商户平台 → **开通「微信电子发票」**
- 实际开票由**开票服务商**完成（微信只负责把开好的票推到客户卡包/给链接）

---

## 三、开票服务商（实际开票的 API）

选并签约一个支持微信电子发票的服务商（**诺诺/百望/高灯**等），拿到 API 凭证。

### 1. 服务商/财务要提供
- API 文档 + `appKey/appSecret/接口地址`（**测试 + 正式**两套）+ 沙箱环境
- 销方信息：公司全称、纳税人识别号、地址电话、开户行账号
- **商品税收分类编码**（维修服务对应编码）+ **税率**（6% 或 13%，以税务核定）

### 2. 云函数环境变量（`cicada-admin-order`）
| 环境变量 | 说明 |
|---|---|
| `INVOICE_PROVIDER` | 服务商标识，如 `nuonuo` / `baiwang` |
| `INVOICE_PROVIDER_APP_KEY` | 服务商 appKey |
| `INVOICE_PROVIDER_APP_SECRET` | 服务商 appSecret |
| `INVOICE_PROVIDER_API_BASE` | 服务商接口地址 |
| `INVOICE_SELLER_TAX_NO` | 销方税号 |
| `INVOICE_TAX_CODE` | 商品税收分类编码 |
| `INVOICE_TAX_RATE` | 税率，如 `0.06` |
| `INVOICE_PUSH_WECHAT` | `1` = 同时推送微信电子发票卡包 |

### 3. 代码接入点（开发对接处，已搭好骨架）
- `cicada-admin-order/index.obj.js` → **`callInvoiceProvider(req)`**：按服务商文档实现「下单开票 + 查结果（拿版式文件URL/号码/日期）+（可选）微信卡包推送」，返回 `{ invoice_no, invoice_date, invoice_url }`。
- 其余闭环已就绪：`issueInvoice` 方法、后台「一键开票」按钮、回填 `invoice_info`、客户端「已开票」展示与复制下载。
- 未配置时「一键开票」会明确提示「开票服务商未配置」，不会误判成功。

---

## 四、一键开票流程（已实现的业务规则）

```
客户在小程序填抬头/税号/邮箱申请开票
  → 财务确认到账（payment_status = paid）   ← ⭐ 强制前置：未到账不允许开票
  → 后台工单抽屉「发票登记」点【一键开票】
  → issueInvoice 校验(有申请/未开过/已到账/有抬头税号/金额>0)
  → callInvoiceProvider 调服务商开票、取号码/日期/链接
  → 回填 invoice_info(已开具) + 记审计日志 + 通知客户
  → 小程序「已开票」显示号码/日期、可复制链接下载
```

> 校验位置：`cicada-admin-order.issueInvoice`。到账判断：`order.payment_status === 'paid'`。

## 五、部署清单
1. 配好 **微信支付** `WX_PAY_*`（client-order + admin-order）→ 重新部署这两个云函数
2. 微信支付商户平台**开通微信电子发票**
3. 签服务商，配 `INVOICE_PROVIDER_*`，在 `callInvoiceProvider` 内实现其 API
4. 重新部署 `cicada-admin-order`、重发 PC 后台 `dist`
5. 验证：报修→报价→支付→确认到账→一键开票→小程序已开票可下载
